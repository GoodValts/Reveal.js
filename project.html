Final project presentation

Team success and key to it

Each member of our team came to the final task:

1. With a clear vision of how they wanted to work on it:

- We didn't want to use frameworks

- Our goal was to achieve a good result and show our skills

2. By taking an active role in team formation

Members similar qualities

1. We want to create a highly stable and easy to use application

2. We are ranked in the top 200

3. We have almost maximum score on cross-checks

4. We show progress in their skills from task to task

5. Each their work is done with passion

6. Everyone has their own strengths that complement the strengths of others

7. Everyone has a primary job, but can dedicate the rest of time to the project

Our organization principles:

1. We start conference calls before the start of each sprint and plan our actions

2. We actively take a part in task discussions and task distribution

3. We don't divide tasks into separate parts; we work together as a team, each on their own component

4. Each member sets their own intermediate deadlines, but remeber that we are a team

5. Most of questions will be resolved in the Discord chat, with additional conference calls conducted if necessary

6. Each team member participates in cross-checks of others' work

Our initial roles:

• Vanya take on the overall project management, configuration, design, styles, as well as deciding the functionality if necessary

• Katya connected the view with all internal logics/functions, actively working on event handlers and forms

• Vladimir primarily worked on the underlying logic of the application

About working with API

1. The team leader initiates the work with the API

2. Other members develop requests and handle responses according to their needs

3. If difficulties arise, other members provide advice and assistance or solve the problem ourselves

4. If changes, corrections, or errors occur, member handles them promptly and resolves any issues
